# 使用文档

## 安装

### 1. 下载

打开 [Releases](https://github.com/Artistkisa/0.6b-Light-novelist-AI-desktop/releases) 页面，下载最新版的 `novelist-desktop-v1.0.zip`。

### 2. 解压

将 zip 文件解压到任意位置，例如 `D:\ novel\`。注意：**路径不要包含中文或特殊符号**，否则 bat 脚本可能出错。

解压后的目录结构：
```
novelist-desktop/
├── T1.gguf                          ← 模型文件
├── llama-b8986-bin-win-cuda-12.4-x64/ ← llama.cpp 服务端
├── webui/                            ← 自定义网页界面
├── start_llama.bat                   ← 启动脚本
└── ...
```

### 3. 启动

双击 `start_llama.bat`，稍等几秒后会自动打开浏览器，地址是 `http://127.0.0.1:8080`。

首次启动时，llama.cpp 需要将模型加载到显存，根据显卡速度可能需要 **5-30 秒**。窗口里会显示加载进度，看到 `using device CUDA0` 就说明 GPU 加速已启用。

## 界面说明

### 侧边栏

- **New Chat**：新建对话，每次新建都会清空上下文
- **对话历史**：点击历史记录可继续之前的对话
- **Settings**（⚙️）：调整生成参数

### 设置项（Settings）

| 设置 | 默认值 | 说明 |
|------|--------|------|
| System Message | 轻小说写手人设 | 定义 AI 的角色和风格。可改为 "翻译助手"、"代码审查员" 等 |
| Temperature | 0.8 | 随机度。越高越有创意，越低越保守 |
| Max Tokens | 1024 | 单次回复的最大长度。0.6B 模型实际通常 300-500 就停 |
| Show Thought | 开 | 是否展开显示 `<think>` 思考过程 |
| Theme | System | 跟随系统/浅色/深色 |

### 聊天界面

- 底部输入框输入提示词，按 **Enter** 发送
- 支持多轮对话，但建议 **每 3-5 轮新建一次对话**，否则上下文过长模型会崩
- 每条 AI 回复右下角有 **复制** 按钮

## 提示词技巧

### 擅长什么

✅ **世界观设定**
```
帮我设计一个蒸汽朋克+魔法的轻小说世界观，包含：
1. 世界背景（时代、地理、政治格局）
2. 魔法系统规则
3. 三个主要势力的冲突
```

✅ **场景描写**
```
写一段雨夜废弃车站的场景，要求有：
- 强烈的视觉意象（锈迹、积水、霓虹倒影）
- 一个神秘角色的出场
- 悬疑氛围
```

✅ **人物设定**
```
设计一个轻小说女主：
- 表面是普通高中生的地下黑客
- 口头禅和标志性动作
- 和男主的初次相遇场景
```

### 不擅长什么

❌ 复杂数学推理
❌ 超长逻辑链（"如果 A 则 B，如果 B 则 C..."）
❌ 搞笑/幽默段子（日式吐槽很生硬）
❌ 严格遵循设定的一致性（写多了会吃书）

### 调优建议

| 想要的效果 | 调整方法 |
|-----------|---------|
| 输出更长 | Temperature 降到 0.6，给更具体的字数要求 |
| 更有创意 | Temperature 提到 1.0-1.2 |
| 减少重复 | Repeat Penalty 提到 1.3-1.5 |
| 日式轻小说味 | 在 System Message 里加 "使用日式轻小说文风，有大量心理描写和拟声词" |
| 中文古风 | System Message 改为 "你是一位古风小说写手，使用文言白话交杂的风格" |

## 故障排除

### 双击 start_llama.bat 后黑窗口闪退

1. 检查 `T1.gguf` 是否在解压后的根目录
2. 检查 `llama-b8986-bin-win-cuda-12.4-x64/llama-server.exe` 是否存在
3. 右键 `start_llama.bat` → 编辑，确认没有乱码

### 浏览器打开后显示 "Connection refused"

模型还在加载中，等 10-30 秒后刷新页面。看黑窗口里有没有 `using device CUDA0` 或 `model loaded`。

### 生成速度很慢（<10 tok/s）

说明在用 CPU 跑。检查：
1. 有没有 NVIDIA 显卡
2. 显卡驱动是否正常
3. 任务管理器 → 性能 → GPU 利用率是否为 0%

如果没有 NVIDIA 显卡，这是正常的。CPU 速度约 40 tok/s（9800X3D），普通 CPU 可能只有 10-20 tok/s。

### 输出全是 `<think>` 内容，没有正文

这是 0.6B 模型的问题。解决方法：
1. 换更简单的提示词（少给限制条件）
2. 把 Temperature 调到 0.9-1.0
3. 直接说 "不要思考，直接写"

### 显存不足（Out of Memory）

llama-server 会自动管理显存。如果显存 < 4GB，模型可能会部分加载到内存，速度下降但应该仍能运行。如果彻底跑不了，只能换 CPU 模式：

编辑 `start_llama.bat`，把 `-ngl 99` 改成 `-ngl 0`，这样所有层都在 CPU 上计算。

### 中文字显示乱码

确保 bat 文件第一行有 `chcp 65001 >nul`（UTF-8 编码）。如果已经存在但仍乱码，可能是 Windows 区域设置问题，去控制面板 → 区域 → 管理 → 更改系统区域设置 → 勾选 "Beta: Unicode UTF-8"。

## 高级：替换模型

如果你有其他 GGUF 格式的模型，可以替换 `T1.gguf`：

1. 把新模型重命名为 `T1.gguf`，覆盖原文件
2. 或者编辑 `start_llama.bat`，把 `-m T1.gguf` 改成 `-m 你的模型.gguf`

注意：不同模型的 chat template 可能不兼容，替换后 WebUI 的 reasoning 框和 system prompt 行为可能异常。

## 隐私说明

- **完全本地运行**，所有对话数据只存在于你的电脑，不会上传到任何服务器
- 关闭黑窗口（llama-server）后，对话历史不会自动保存（除非你手动导出）
- 模型权重和代码均开源，可审计
